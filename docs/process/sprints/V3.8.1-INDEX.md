# Wave V3.8.1 — Correções pós-V3.8 (input do hotplug, GUI saudável, modo jogo)

Índice da wave V3.8.1. Motivação: durante o review de UI/UX da v3.8 instalada na máquina, a
mantenedora reportou problemas concretos que a wave anterior não cobria — drift dos sticks no
hotplug do controle, lentidão geral da GUI, dropdowns ilegíveis no COSMIC e ausência de uma
forma rápida de desativar a emulação de mouse/teclado ao entrar num jogo.

A sessão diagnosticou os problemas (incluindo um busy-loop a 100% CPU que crescia 5+ GB de RAM
em minutos), corrigiu cada um com commits atômicos e fechou com release `v3.8.0 → v3.8.1`.

## Sprints

| ID | Sprint | Entrega |
|----|--------|---------|
| 140 | BUG-DAEMON-EVDEV-HOTPLUG-CACHE-01 | sticks lidos certos quando o controle conecta após o boot do daemon (re-procura o evdev a cada `connect()`, em vez de cachear `_device_path=None` no `__init__`) |
| 141 | PERF-GUI-PROFILE-LOAD-NONBLOCKING-01 | aba Perfis não trava mais ao clicar/digitar/salvar — `load_all_profiles()` virou worker thread + cache em memória (`_profiles_cache`) consultado por `on_profile_selection_changed` e `_build_profile_from_editor` |
| 142 | BUG-GUI-COMBOBOX-POPUP-SELECTED-COSMIC-01 | item atualmente selecionado do dropdown agora respeita o tema Drácula (CSS cobre `:selected`/`:active`/`:checked` do `menu menuitem` em paridade com `:hover`/`:focus`) |
| 143 | FEAT-EMULATION-GAMEMODE-LONGPRESS-01 | modo jogo via long-press do PS (~1s): suprime só a emulação de mouse/teclado mantendo hotkeys ativos; notifica via D-Bus; expõe `emulation_suppressed` em `daemon.status`/`state_full` + método IPC `daemon.emulation.suppress` (toggle/set) |
| 144 | BUG-GUI-IDLE-ADD-BUSY-LOOP-01 | **104% CPU → 2.4%, 5.3 GB → 90 MB**: `install_status_polling` passava `_tick_live_state`/`_tick_profile_state` (callbacks que retornam `True` pro `timeout_add`) direto para `GLib.idle_add`, criando dois busy-loops infinitos. Wrappers one-shot (`lambda: fn() and False`) corrigem |

## Resultado

- Release `v3.8.1` no fork (`[REDACTED]/hefesto-dualsense4unix`) com 6 assets: AppImage,
  Flatpak, dois `.deb` (py310/py312), wheel e sdist.
- Suíte unit: **1450 testes verdes** (+32 vs. v3.8.0).
- CI: 3min33s, success.
- Validação na máquina: daemon ativo + IPC respondendo + sticks centrados em 128/128/128/128;
  `emulation_suppressed=False` exposto no `state_full`.

## Notas de retrospectiva

- **`pgrep -f` mata o próprio shell** quando o pattern aparece no comando — armadilha real durante
  a sessão (já era memória, reforçada com a derrubada acidental do daemon ao filtrar pelo `comm`
  truncado `hefesto-dualsen`).
- **`py-spy dump --pid` foi decisivo** para achar o busy-loop. Memória registrada para reuso em
  qualquer sessão futura com GUI Python lenta (`reference-py-spy-dump-gui-perf`).
- **Auditoria do `uninstall.sh`** revelou um bug pré-existente: `sudo` dentro do script falha
  silenciosamente sem cache de TTY, deixando o applet COSMIC residual em `/usr/local/bin` e
  `/usr/share/applications`. Anotado para wave futura (não bloqueia v3.8.1; a correção do
  `--purge-config` aplicada nesta wave removeu os glyphs orfanados).
