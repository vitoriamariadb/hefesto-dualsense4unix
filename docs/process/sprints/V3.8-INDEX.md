# Wave V3.8 — Controle de ativação, robustez (doctor everywhere) e applet visível

Índice da Wave V3.8 (sprints 130–139). Motivação: após a v3.7.0, a mantenedora pediu para
generalizar a "sacada do doctor" para as demais features, poder desativar o programa sem
desinstalar, deixar o daemon mais seguro e inteligente, e corrigir o applet que não aparecia na
guia de Miniaplicativos do COSMIC.

## Sprints

| ID | Sprint | Entrega |
|----|--------|---------|
| 130 | BUG-COSMIC-APPLET-PANEL-VISIBILITY-01 | applet aparece em Miniaplicativos (X-HostWaylandDisplay + PNG 256 + `killall cosmic-panel`) |
| 131 | FEAT-DAEMON-PAUSE-RESUME-01 | pausar/retomar input em runtime (CLI + IPC + applet); daemon vivo sem afetar o sistema |
| 132 | FEAT-DAEMON-DISABLE-CONTROL-01 | `daemon disable`/`enable` — desligar sem desinstalar |
| 133 | FEAT-DOCTOR-CLI-AND-CHECKS-01 | `hefesto-dualsense4unix doctor` no CLI + checks do daemon via IPC |
| 134 | FEAT-DAEMON-RESILIENT-SUBSYSTEMS-01 | um subsystem que falha no boot não derruba o daemon |
| 135 | FEAT-IPC-REQUEST-VALIDATION-01 | anti-regressão da resiliência do dispatcher IPC (infra já era robusta) |
| 136 | FEAT-CONFIG-AUDIT-BOOT-01 | audita perfis no boot e avisa sobre corrompidos |
| 137 | FEAT-DAEMON-GRACEFUL-SHUTDOWN-01 | fecha IPC/UDP com timeout no shutdown (já era gracioso) |
| 138 | FEAT-SYSTEM-AUTOREPAIR-BOOT-01 | detecta infra quebrada no boot (udev/WirePlumber) e avisa o reparo |
| 139 | DOCS-V3.8-ROOT-AND-ORG-01 | docs da raiz (testes, troubleshooting com doctor/pausar) + índice + branch |

## Níveis de "desativar" (resultado da wave)

pausar (runtime, daemon vivo) → desligar (`disable`: para + sem auto-start) → desinstalar (`purge`).

## Notas

- Robustez "no espírito do doctor": auditoria de config no boot (136), detecção + aviso de infra
  (138), resiliência de subsystems (134) e do shutdown (137). Em 135 e 137, a auditoria mostrou que
  a infra já era madura — o escopo foi ajustado para anti-regressão + o gap real (timeout), sem
  reescrita redundante. Honestidade > inflar diff.
- Follow-up: GUI button de pausar (a parte visível ficou no applet COSMIC + CLI) e checks extras do
  doctor (portas UDP/metrics, saúde por subsystem).
